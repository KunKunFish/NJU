## 三、已完成的研究工作

### 3.1 文献研究与理论梳理

在开题报告文献综述的基础上，本阶段进一步扩大了文献检索范围，累计阅读中英文文献60余篇，其中近五年文献占比超过70%。文献研究主要围绕以下三个方向展开：

（1）充电设施选址优化方面：系统梳理了公共设施选址理论的发展脉络，重点研究了层次分析法（AHP）、熵权法、TOPSIS法、模糊综合评价法等在设施选址领域的应用案例，明确了多准则决策方法在充电桩选址中的适用性与局限性。

（2）成本建模方法方面：深入研究了全生命周期成本（LCC）理论、作业成本法（ABC）以及回归分析在基础设施成本预测中的应用，为后续安装成本与维护成本模型的构建提供了方法论支撑。

（3）多城市比较研究方面：梳理了国内外关于不同城市充电基础设施差异化布局的文献，总结了城市属性（经济水平、人口密度、政策环境等）对充电设施建设成本和运营效率的影响机制。

在理论框架方面，本研究最终确定以公共设施选址理论为基础，融合消费者行为理论和成本收益理论，构建"选址因素—成本结构—综合决策"三位一体的研究框架。

### 3.2 C公司运营现状调研

本人作为C公司管理者，直接参与公司充电桩业务的运营管理工作，能够便捷地获取公司内部一手数据资料。本阶段通过内部运营系统数据导出、财务台账梳理及与各地区运维团队沟通等方式，完成了三地充电桩运营数据的收集与整理工作。具体包括：

（1）数据收集情况：从公司内部管理系统中导出南京、深圳、徐州三地近一年的运营数据，涵盖站点基本信息（位置、规模、安装时间）、使用数据（日均充电次数、高峰时段分布、设备利用率）、财务数据（安装成本明细、月度营收、维护支出）以及用户反馈数据（投诉记录、满意度调查）。

（2）运营经验梳理：结合本人在公司业务拓展过程中积累的实践经验，对现有选址决策流程、成本管控痛点及扩张规划进行了系统性总结与反思。

（3）现状分析结论：
- 三地共计运营站点若干个，覆盖社区类型包括老旧小区、新建商品房小区及混合型社区；
- 设备平均利用率存在明显的城市间差异与社区间差异；
- 现有选址主要依赖运营人员经验判断，缺乏标准化的评估工具；
- 安装成本在三地之间存在20%—40%的波动，主要受人工费用、电力改造难度及物业协调成本影响。

### 3.3 选址影响因素体系构建

基于文献研究与企业调研，本研究完成了选址影响因素的系统识别与权重计算，具体工作如下：

（1）因素识别：通过文献分析、专家访谈与实际站点评估，最终确定了四大类、十余项关键选址影响因素：

- 需求侧因素：电动自行车保有量与密度、用户充电习惯与高峰时段、目标区域人口密度；
- 供给侧因素：场地可获得性与租金、电力接入条件与成本、周边竞争充电桩情况；
- 环境与政策因素：消防安全条件、区域规划与政策支持、交通便利性；
- 技术与运营因素：设备安装便利性、物业协同程度、网络信号覆盖。

（2）权重计算：采用AHP法与熵权法相结合的组合赋权方式。AHP法通过邀请C公司管理人员及行业专家进行两两比较判断，获取主观权重；熵权法基于三地实际运营数据，利用信息熵计算客观权重。最终通过线性加权法将主客观权重进行综合，形成最终的因素权重矩阵。

（3）初步结论：权重分析结果表明，用户需求密度、电力接入条件和物业协同程度是影响选址成效的前三位关键因素，合计权重占比超过45%。

### 3.4 成本结构分析初步成果

（1）安装成本结构模型：对C公司三地的安装成本进行了系统拆解，将其分为六大类：硬件设备采购、基础施工（土建与布线）、电力接入改造、通信模块安装、验收与手续费用、物业协调费用。通过三地数据对比，识别出人工成本系数、电力接入系数和市政施工系数是导致城市间安装成本差异的三大关键驱动因素。

（2）维护成本数据整理：完成了三地历史维保记录的清洗与分类，将维护成本划分为设备故障维修、损耗件更换、通信模块维护、定期巡检及物业协调五个子类。初步统计显示，设备故障维修占维护总成本的比例最高，约为35%—45%，其次为损耗件更换和定期巡检费用。

（3）三地成本差异关键发现：
- 深圳的安装成本显著高于南京和徐州，主要受人工费用和场地改造成本驱动；
- 徐州的维护成本相对较低，与设备使用强度较低和物业配合度较高有关；
- 南京的综合成本处于中等水平，但运营收益表现较为稳定。

## 四、初步研究成果

经过前期的数据收集、因素分析与模型构建工作，本研究已取得以下阶段性成果：

### 4.1 选址影响因素权重矩阵

通过AHP法与熵权法的组合赋权，形成了社区电动自行车充电桩选址影响因素的综合权重体系。主要结论如下：

| 一级指标 | 权重 | 二级指标（前两位） | 权重 |
|----------|------|-------------------|------|
| 需求侧因素 | 0.35 | 电动自行车保有量与密度 | 0.15 |
|  |  | 用户充电高峰时段匹配度 | 0.11 |
| 供给侧因素 | 0.28 | 电力接入条件与成本 | 0.13 |
|  |  | 场地可获得性与租金 | 0.09 |
| 环境与政策因素 | 0.20 | 消防安全条件 | 0.09 |
|  |  | 区域政策支持力度 | 0.06 |
| 技术与运营因素 | 0.17 | 物业协同程度 | 0.08 |
|  |  | 设备安装便利性 | 0.05 |

该权重矩阵已通过一致性检验（CR<0.1），可作为后续综合评价模型的输入基础。

### 4.2 安装成本结构模型初步成果

基于三地实际项目数据，构建了安装成本的标准化拆解框架，各成本子项占比如下：

| 成本子项 | 南京占比 | 深圳占比 | 徐州占比 |
|----------|----------|----------|----------|
| 硬件设备采购 | 约40% | 约35% | 约45% |
| 基础施工（土建与布线） | 约20% | 约25% | 约18% |
| 电力接入改造 | 约18% | 约20% | 约15% |
| 通信模块安装 | 约8% | 约7% | 约9% |
| 验收与手续费用 | 约5% | 约5% | 约5% |
| 物业协调费用 | 约9% | 约8% | 约8% |

初步建立了以城市属性系数（人工成本系数、电力接入系数、市政施工系数）为核心变量的安装成本估算公式，可实现对新城市安装成本的快速预估。

### 4.3 维护成本分类与规律性发现

对三地历史维保数据的初步统计分析表明：
- 设备故障维修费用与设备运行时长呈正相关，安装满12个月后故障率明显上升；
- 损耗件更换具有周期性特征，平均更换周期约为8—12个月；
- 物业协调费用在老旧小区显著高于新建小区；
- 维护成本整体呈现前低后高的趋势，符合设备"浴盆曲线"的早期磨合与老化阶段规律。

上述发现为后续构建全生命周期维护成本预测模型提供了数据支撑。

### 4.4 综合评价模型框架搭建

在整合选址因素权重与成本结构的基础上，初步搭建了综合评价模型的框架，拟采用TOPSIS法对候选站点进行排序。模型输入包括：选址因素得分（基于权重矩阵）、预估安装成本、预估维护成本及预期投资回收周期。目前模型框架已确定，正在进行数据代入与结果验证工作。

## 五、研究中遇到的问题与困难

### 5.1 数据获取方面

（1）部分历史数据缺失：公司早期站点在建设初期未建立完善的数据记录体系，部分站点的安装成本明细、早期维修记录存在缺失或记录不规范的情况，增加了数据清洗和补全的工作量。

（2）维护数据颗粒度不够细：部分维保记录仅记录了总费用，未对故障类型、更换部件、工时等进行细分，对维护成本的结构化拆解造成一定困难。

（3）用户行为数据有限：充电桩后台系统记录了充电频次与时段分布，但缺乏用户满意度、选择偏好等主观评价数据，需要通过其他渠道补充。

### 5.2 方法应用方面

（1）AHP主观赋权的专家样本有限：由于电动自行车充电桩行业相对细分，能够提供专业判断的行业专家数量有限，AHP判断矩阵的构建主要依赖公司内部管理团队的经验，可能存在一定的主观偏差。

（2）维护成本预测模型的样本量约束：三地站点的运营时间普遍在一到两年内，历史维保数据的时间跨度有限，对构建长周期预测模型的精度产生一定影响。

（3）跨城市可比性处理难度：三地在经济发展水平、政策环境、物业管理模式等方面差异较大，如何将这些非量化因素合理纳入模型是一个需要持续优化的问题。

### 5.3 其他困难

本人在职攻读学位，需兼顾公司日常运营管理与论文研究工作，时间分配上存在一定压力。但同时，直接参与公司业务运营也为研究提供了持续的数据获取渠道和实践验证机会。

## 六、对原研究计划的调整

### 6.1 调整内容

对照开题报告中的原始研究计划，本阶段在以下方面进行了适度调整：

（1）维护成本模型方法调整：原计划采用时间序列分析或随机森林回归构建维护成本预测模型。由于目前数据时间跨度有限，时间序列方法的适用性受到约束，拟调整为以多元回归分析为主、结合设备老化经验公式进行建模，待后续数据积累充分后再考虑引入更复杂的机器学习方法。

（2）外部验证范围调整：原计划在条件允许时选取新城市进行模型外推验证。考虑到研究时间和资源限制，拟将验证方式调整为：以三地中某一城市的部分站点数据作为验证集进行交叉验证，而非新增城市的全面验证。

（3）论文章节侧重点微调：在撰写过程中发现，社区电动自行车充电桩与汽车充电桩在选址逻辑上差异较大，现有文献的直接参考价值有限。因此适当增加了第三章（C公司运营现状分析）的篇幅，以加强对电动自行车充电桩独特运营场景的描述与分析。

### 6.2 调整原因

上述调整主要基于研究推进过程中对数据条件、方法适用性的实际认知深化，并未改变研究的总体目标与核心框架。调整后的技术路线更加贴合企业实际数据条件，有助于确保研究成果的可靠性与实用性。
